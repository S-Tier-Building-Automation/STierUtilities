<Note>Provided by **BACnet Historian** — contract v1.0 — Service</Note>

Continuously logs selected BACnet points into the timeseries service on a scheduler cadence — a composed capability built from bacnet.read + scheduler + timeseries.

## Consume it

```js
// 1. Declare it in your tool manifest:
requires: [{ capability: "bacnet.historian", version: "^1.0" }]

// 2. Resolve it from your scoped host and call it:
const bacnetHistorian = host.use("bacnet.historian.v1");
const result = await bacnetHistorian.addPoint(…);
```

## Methods

### `addPoint({ device, objectType, instance, label? })`

Returns `PointRecord`.

Add a point to historize (deduped by device:type:instance).

### `removePoint(point)`

Returns `boolean`.

Stop historizing a point.

### `points()`

Returns `PointRecord[]`.

Current points with last value/error/read count.

### `pollOnce()`

Returns `Promise<{ written, errors, points }>`.

Read every point once and write values to timeseries.

### `start(intervalMs = 60000)`

Returns `void`.

Begin logging on a schedule (polls immediately).

### `stop()`

Returns `void`.

Stop the scheduled logging.

### `isRunning()`

Returns `boolean`.

Whether scheduled logging is active.

## Provider permissions

The provider holds these platform permissions on your behalf:

`timeseries.write` · `scheduler.register`
