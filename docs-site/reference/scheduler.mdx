<Note>Provided by **Observability** — contract v1.0 — Service</Note>

Shared recurring-job runner. Register work by id + interval; the scheduler runs it, skips overlapping ticks, and tracks run counts and the last error.

## Consume it

```js
// 1. Declare it in your tool manifest:
requires: [{ capability: "scheduler", version: "^1.0" }]

// 2. Resolve it from your scoped host and call it:
const scheduler = host.use("scheduler.v1");
const result = await scheduler.register(…);
```

## Methods

### `register(id, { intervalMs, run, immediate? })`

Returns `string`.

Register (or replace) a recurring job. `run` is your async work; `immediate:true` runs one tick right away.

### `unregister(id)`

Returns `boolean`.

Stop and remove a job. Returns false if no such job.

### `has(id)`

Returns `boolean`.

Whether a job id is currently registered.

### `runNow(id)`

Returns `Promise<void>`.

Run a registered job's tick once, now (awaitable).

### `list()`

Returns `Array<{ id, intervalMs, runs, lastError, running }>`.

Snapshot of all registered jobs.
